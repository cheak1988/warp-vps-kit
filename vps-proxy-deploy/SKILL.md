# VPS Proxy Deploy

Use this skill to guide a user through rescuing a cheap VPS that still runs but is poor or impossible to reach directly. The core idea is:

```text
Client -> Cloudflare CDN -> VPS Xray VLESS+WS -> WARP SOCKS5 proxy -> Internet
```

This does not unblock the VPS IP. It changes the client ingress path so users no longer connect to the origin IP directly. If Cloudflare can still reach the origin, a direct-connect-broken VPS may become usable again.

## First Response

Start by checking what the user already has:

- VPS provider, OS, and whether VNC console is available.
- Whether the VPS still boots and can be managed somehow.
- Domain on Cloudflare.
- Desired client: Clash Verge, V2RayN, or both.
- Whether the goal is rescue, speed improvement, first-time setup, or troubleshooting.

Do not ask for real passwords or Cloudflare API tokens unless absolutely necessary. Prefer placeholders and local commands the user can run.

## Workflow

1. Explain the architecture and limits in plain language.
2. Collect non-secret parameters:
   - `domain`, for example `example.com`
   - `proxy_subdomain`, usually `ws`
   - `config_subdomain`, usually `cfg`
   - `vps_host`, the VPS origin IP or hostname
   - `vless_uuid`, generated by `xray uuid`
   - `ws_path`, usually `/ws`
3. Generate configs with `scripts/render_config.py`.
4. Instruct the user to create a Cloudflare DNS A record for `ws.example.com` pointing to the VPS origin with orange-cloud proxy enabled.
5. Use Worker config hosting when VNC or copy/paste is unreliable.
6. Install Xray VLESS+WS and WARP proxy mode on the VPS.
7. Import the generated Clash Verge or V2RayN config.
8. Run `scripts/doctor.py` and troubleshoot from `references/troubleshooting.md`.
9. Run `scripts/redact.py` before the user shares logs, configs, screenshots, or benchmark output.

## Bundled Scripts

- `scripts/render_config.py`: generate Xray config, Worker JS, Clash Verge YAML, V2RayN link, and next steps.
- `scripts/doctor.py`: validate config shape and optionally check DNS/TCP/WebSocket.
- `scripts/redact.py`: redact likely real IPs, UUIDs, tokens, and passwords.
- `scripts/install-xray-ws.sh`: VPS-side Xray VLESS+WS installer; defaults to dry run unless `RUN=1`.
- `scripts/install-warp-proxy.sh`: VPS-side WARP proxy-mode installer; defaults to dry run unless `RUN=1`.
- `scripts/doctor-vps.sh`: VPS-side service and WARP status checks.

Prefer these scripts over hand-writing large JSON/YAML blocks.

## Critical Rules

- `ws.example.com` must be Cloudflare-proxied if the goal is to avoid direct client connections to the VPS IP.
- WARP must use proxy mode, not full-tunnel mode, for this architecture.
- Treat `PermitRootLogin yes` and password SSH as rescue-only settings.
- Do not promise guaranteed 4K, permanent availability, or universal rescue.
- Never publish real IPs, UUIDs, passwords, or Cloudflare API tokens.
